How the system currently flows (request lifecycle)
1. `server.js` boots: loads env, attempts MongoDB and Redis connections, then starts HTTP server.
2. `app.js` configures middleware: Helmet, logging (morgan), CORS, JSON parser, rate limiter, and registers routes.
3. Incoming request hits a route (auth, business, leads, health).
4. Protected routes use `auth.middleware.js` which verifies JWT and attaches `req.user` and `req.currentUser` (from DB).
5. Route handlers call services and models; e.g., `AuthService` performs user creation, password hashing (bcryptjs), and token generation (JWT). Refresh tokens are persisted to Redis.
6. Responses use the project's standardized JSON shapes for success and error.
